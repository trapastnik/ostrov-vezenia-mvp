# Changelog

Все значимые изменения в проекте «Остров Везения» документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
проект придерживается [Semantic Versioning](https://semver.org/lang/ru/).

---

## [0.3.0] — 2026-03-10

### Безопасность
- Полный аудит безопасности в 3 фазы: критические уязвимости, усиление защиты, шифрование PII
- Шифрование паспортных данных (Fernet AES-128) с маскированием при выводе
- Аудит-лог всех действий операторов (`audit_logs`)
- CHECK-ограничения на уровне БД для статусов заказов, партий, деклараций
- CSP, X-Frame-Options, HSTS, Rate Limiting — все security-заголовки настроены
- Валидация паспортных серий/номеров, телефонов, адресов на бэкенде

### Добавлено
- **Партии (Batches):** UI создания партии из заказов, детальная страница партии с каскадом статусов (#42, #43)
- **Группы отправок:** автоматическая группировка заказов по хабам с оптимизацией стоимости
- **Справочник ТН ВЭД:** полный импорт 13K+ кодов, браузер дерева, поиск по кодам
- **Вебхуки:** уведомления магазинов о смене статусов заказов (HMAC-SHA256)
- **Отправки Почты России:** создание почтовых отправлений через API
- **Трекинг:** обновление статусов из API Почты + отслеживание в Bitrix
- **Привязка партия ↔ декларация:** автоматическое создание ДТЭГ при создании партии
- **Версионность продукта:** страница истории версий в админ-панели

### Изменено
- Миграция ПТД-ЭГ → ДТЭГ (Решение ЕЭК №142)
- PDF-превью деклараций: `Content-Disposition: inline` + `<iframe>` для надёжного рендеринга
- Rate limiter: graceful fallback на in-memory при недоступности Redis

### Исправлено
- PDF-превью не отображалось в iframe (CSP `frame-src blob:`, `object-src blob:`)
- Отмена заказа вызывала 500 (uuid → varchar(36) в PostgreSQL)
- Пустой список заказов на проде (отсутствовала колонка `batch_id`)
- Спам предупреждений PII_ENCRYPTION_KEY (singleton с `_fernet_checked`)
- Аудит-лог терялся (добавлен commit после `log_action`)

## [0.2.1] — 2026-03-03

### Добавлено
- **Модуль таможенных деклараций:** CRUD деклараций, валидация, экспорт CSV/PDF
- **PDF-просмотр:** встроенный просмотр ДТЭГ прямо в админ-панели
- **Редактор таможенных полей:** ТН ВЭД, страна происхождения, бренд — прямо в карточке заказа
- **ДТЭГ compliance:** лимит 200 EUR / 31 кг, курсы ЦБ РФ, паспортные данные
- CI/CD: GitHub Actions автодеплой при push в main
- Блок «Подключение к серверу» на странице System Health

### Исправлено
- JSON field change detection (SQLAlchemy `flag_modified`)
- Шрифт DejaVuSans для кириллицы в PDF

## [0.2.0] — 2026-02-21

### Безопасность
- Первичный аудит безопасности: CORS whitelist, JWT expiry, rate limiting
- Security headers в nginx: X-Frame-Options, X-Content-Type-Options, HSTS
- Закрыты /docs и /openapi.json на проде (только localhost)
- Rate limiter: Redis storage, X-Forwarded-For для реального IP

### Добавлено
- **System Health:** мониторинг сервисов, тесты системы, метрики сервера
- **Тарификатор:** сравнение публичных и контрактных тарифов Почты России
- **Группировка отправок:** алгоритм оптимальной консолидации по хабам
- **Трекинг:** собственная система отслеживания посылок
- **Панель отладки API:** raw request/response для запросов к Почте России
- Версионность проекта (pyproject.toml + package.json)

### Исправлено
- CORS_ORIGINS — str с @property вместо list[str]
- Circular import limiter → вынесен в app/core/limiter.py
- UUIDType совместимость с asyncpg UUID

## [0.1.0] — 2026-02-17

### Добавлено
- **MVP:** Backend API (FastAPI), админ-панель (Vue 3), эмулятор магазина
- Заказы: создание через X-API-Key, просмотр, смена статусов
- Магазины: CRUD с API-ключами
- Расчёт стоимости доставки через API Почты России
- JWT авторизация для операторов
- Docker Compose для dev и prod
- Nginx reverse proxy
- Bitrix-плагин: базовая структура (ApiClient, OrderMapper, SaleEvents)
- Документация: архитектура, API, БД, деплой
